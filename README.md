 AI Product Experiments

This repository contains small experiments exploring how AI can improve product workflows, automation, and decision making.

The goal is not building full products, but quickly testing ideas that could improve real SaaS systems.

---

 Experiment 1 – AI Support Assistant

Problem

Customer support teams spend significant time answering repetitive questions.

Idea

Use an AI assistant trained on product documentation to answer common support questions instantly.

Possible Benefits

- Faster response time
- Reduced support workload
- Better customer satisfaction

Metrics to Track

- First response time
- Support ticket volume
- AI resolution rate

---

 Experiment 2 – AI Feature Prioritization

Problem

Product teams often struggle to prioritize features based on scattered user feedback.

Idea

Use AI to analyze feedback from multiple sources:
- support tickets
- user reviews
- surveys

AI clusters feedback and highlights the most requested features.

Expected Impact

- clearer roadmap decisions
- better alignment with user needs

---

 Experiment 3 – AI Onboarding Optimization

Problem

Many SaaS users drop during onboarding.

Idea

AI analyzes user behavior and suggests improvements in onboarding steps.

Example

If users frequently drop at step 3, AI suggests simplifying that step.

Key Metric

- onboarding completion rate
